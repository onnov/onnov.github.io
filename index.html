📘 GitHub Pages — инструкция

Скачать инструкцию (Markdown)

Инструкция по публикации статического сайта на GitHub Pages. Подходит для личных страниц и проектов. Описаны два способа.

Способ 1. Deploy from a branch (встроенный)

Для статики без сборки

  1. Подготовьте файлы — убедитесь, что index.html находится в корне репозитория.
  2. Запушьте изменения: 
    git add .
git commit -m "Initial commit"
git push origin main
  3. Включите GitHub Pages:
    • Зайдите в Settings → Pages.
    • В разделе Build and deployment → Source выберите Deploy from a branch.
    • Выберите ветку main и папку / (root).
    • Нажмите Save.
  4. Дождитесь публикации:
    • После сохранения GitHub запустит процесс сборки.
    • Перейдите на вкладку Actions — вы увидите workflow “pages build and deployment”.
    • Дождитесь, когда рядом загорится зелёный кружок (успех).
    • Обновите страницу Settings → Pages — вверху появится ссылка на сайт.
  5. Проверьте сайт — откройте ссылку в режиме инкогнито (Ctrl+F5).

Способ 2. GitHub Actions (гибкий)

Для проектов со сборкой или расширенной настройкой

  1. Подготовьте файлы — разместите index.html в корне (или укажите другой путь в workflow).
  2. Создайте workflow-файл:
    • В корне создайте папку .github/workflows.
    • Внутри создайте файл deploy.yml с содержимым: 
      name: Deploy to GitHub Pages

on:
  push:
    branches: [ "main" ]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Setup Pages
        uses: actions/configure-pages@v4
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: '.'
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
      Если index.html лежит в подпапке (например, public), замените path: '.' на path: 'public'.
  3. Запушьте изменения: 
    git add .
git commit -m "Add GitHub Pages workflow"
git push origin main
  4. Настройте источник Pages:
    • Зайдите в Settings → Pages.
    • В Source выберите GitHub Actions.
    • Нажмите Save (если уже выбрано, пропустите).
  5. Запустите деплой:
    • Перейдите на вкладку Actions.
    • Вы увидите workflow Deploy to GitHub Pages.
    • Дождитесь зелёной галочки.
  6. Найдите ссылку на сайт:
    • Вверху Settings → Pages или в логах workflow (шаг Deploy to GitHub Pages).
    • Откройте ссылку в инкогнито.

❓ Возможные проблемы и их решение

ПроблемаРешение
404 Not Found 1. Проверьте источник в Pages (для способа 1 — Deploy from a branch, для способа 2 — GitHub Actions).
2. Убедитесь, что index.html в корне (или в папке, указанной в path).
3. Очистите кеш браузера или откройте в режиме инкогнито.
Битый / не открывающийся файл Создайте новый index.html с минимальным содержимым (см. ниже) и запушьте заново.
Workflow не запускается Проверьте, что файл .github/workflows/deploy.yml лежит в ветке main и назван правильно.
Сайт не обновляется Подождите 2–3 минуты (CDN-кеш). Сделайте жёсткую перезагрузку (Ctrl+F5).

🧩 Минимальный рабочий index.html

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>My Site</title>
</head>
<body>
    <h1>Hello, world!</h1>
</body>
</html>
